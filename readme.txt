1. Сервис доступа к данным BooksApi.

Состоит из двух проектов: 
- BooksApi (WebApi)
- BooksApiClient (Library). BooksApiClient также используется в десктопном приложении.

BooksApi содержит скрипты install.bat, start.bat, stop.bat, uninstall.bat, 
чтобы установить/запустить/остановить/удалить проект как сервис (можно предварительно опубликовать для удобства).
Также проект можно запустить просто как приложение для теста. Порты настраиваются в appsettings.json.
REST сервис для работы с книгами. SQLite база данных. Будет создана в каталоге %ApplicationData%\Books\.
Изображения хранятся отдельно в том же каталоге. Вопросы сжатия изображений для предпросмотра не рассматривались.
Логирование не рассматривалось. Доступ к конексту БД - scoped, хотя для различных background-task это совсем плохо.
При запуске добавлена автоматическая миграция. Обработка ошибок минимальна.

2. Books - WPF клиент. Ссылается на BooksApiClient.
Можно указать адрес подключения.
Можно сгенирировать и добавить случайную книгу (изображения берет из ресурсов рандомно).
Можно добавить книгу, удалить, отредактировать.
Минимальный pagination и фильтрация.
Сортировка по автору.
